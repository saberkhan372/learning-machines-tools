# Video Test Report

Use this with Temporal Telephone, curated video examples, or optional video-generation tests.

Participation options:

- Use: test a video model directly if available and desired.
- Observe / Critique: analyze curated clips without generating video.
- Teach / Design: create a safe classroom investigation of temporal coherence.
- Build / Code: make a frame or motion explainer.
- Critical / No-AI: write a policy, consent protocol, or failure-analysis guide.

## Failure Categories

Mark any that appear:

- Identity drift
- Body distortion
- Hand or finger glitches
- Physics break
- Background teleport
- Object melting
- Camera inconsistency
- Text corruption
- Temporal jump
- Style drift

## Motion Default Test

- What movement does the model assume?
- What camera language does it default to?
- What bodies, gestures, environments, and styles appear?
- What happens when the prompt is vague?
- What breaks over time?

## Reflection

- Human: what did you choose, frame, select, revise, or reject?
- Machine: what did the system generate, maintain, drift, or distort over time?
- System: what data, defaults, interface choices, or platform assumptions shaped the result?
- Ethics: who might be copied, misrepresented, erased, stereotyped, or excluded?
- Pedagogy: how would this need to change for students or community learners?
- What was easier in images than video?
- What kind of motion did the model handle well?
- What broke across time?
- What did I need to anchor?
- What did human editing contribute?
- What ethical concern did this raise?
