## Notion Landing Page (long — full program description)

### Learning Machines: Text, Images, Video

**A CC Fest Creative AI Camp**

---

**Learning Machines: Text, Images, Video** is a beginner-friendly creative AI camp for educators, artists, students, and curious learners. Across three hands-on sessions and one showcase, participants investigate how generative AI systems create text, images, and video.

Each session asks the same question:

> **What is the machine actually doing?**

This is not a tool-promotion workshop. It is not a prompt-engineering course. Participants investigate models as systems — testing behavior, documenting evidence, naming defaults and failures, and reflecting on the role of human judgment.

---

#### The Intellectual Spine

> Text is sequential. Images are spatial. Video is spatiotemporal.

The same basic logic — prediction from learned patterns — produces different results and different failure modes in each modality. Understanding that logic, and what it cannot do, is the point of the camp.

---

#### Sessions

| Session | Focus | Core Question |
| --- | --- | --- |
| 1 | Text | How do language models generate text that feels meaningful? |
| 2 | Images | What is an image model actually working with when it sees or generates? |
| 3 | Video | What changes when generation has to work across time? |
| 4 | Showcase | What did we learn by making, testing, critiquing, or teaching these systems? |

---

#### What Participants Do

Each content session follows the same pattern:

1. **Unplugged** — experience the concept before touching a tool
2. **Mechanism** — an interactive tool makes the hidden process visible
3. **Investigation** — a structured A/B/C prompt test, Default Test, or failure analysis
4. **Evidence** — document prompts, outputs, observations, failures, and revisions
5. **Reflection** — Human / Machine / System / Ethics / Pedagogy

---

#### Participation Pathways

Every activity has five pathways. Opting out of direct AI use never means opting out of the camp.

| Pathway | What It Means |
| --- | --- |
| Use | Try the AI tool directly and document what happens |
| Observe / Critique | Analyze pre-generated examples without logging into a tool |
| Teach / Design | Create a lesson, worksheet, or facilitation plan |
| Build / Code | Make an explainer or small interactive tool |
| Critical / No-AI | Create a critique, consent checklist, model card, policy, or unplugged activity |

---

#### Final Project Pathways

Participants choose one for the Session 4 showcase:

| Pathway | Final Artifact |
| --- | --- |
| Make | Creative text/image/video artifact with process evidence |
| Teach | Classroom-ready activity, handout, or facilitation plan |
| Investigate | Model behavior experiment or Default Test |
| Explain | Interactive tool, poster, zine, or concept bridge |
| Critical / No-AI | Critique, consent checklist, model card, policy, or unplugged activity |

---

#### Ethics and Consent

Ethics is structural, not topical. It appears in every session, every tool, and every final project.

- AI use is optional, visible, and discussable.
- Model defaults are treated as evidence of data, design, culture, and platform decisions.
- Public recaps and screenshots require consent and human review.
- We name labor, consent, data, and bias concerns throughout — not just at the end.

---

#### Tools

All tools are single-file HTML, offline-friendly, and hosted on GitHub Pages. No accounts required.

- **Tokenizer + Temperature Visualizer** — makes tokenization, probability, and temperature visible (Session 1)
- **ELIZA Simulator** — makes rule-based AI legible and contrasts it with LLM prediction (Session 1)
- **Diffusion Step-Through Viewer** — makes iterative denoising visible with a slider (Session 2)
- **Feature Extraction / Pixel Resolution Tool** — makes image representation visible (Session 2)
- **Temporal Telephone** — makes frame-by-frame drift and anchoring visible (Session 3)
