# Session 1 Facilitation Notes: Text

## Session Goal

Participants investigate how text generation can feel meaningful even when the visible mechanism is tokenization, probability, and pattern continuation.

Core question: when a language model writes, what is it actually doing?

## Tool Order

1. Unplugged prediction game: participants guess next words from sentence stems.
2. Tokenizer + Temperature Visualizer: show tokens, probability bars, greedy decoding, sampling, and temperature.
3. ELIZA Simulator: compare transparent rule matching with more fluent but less inspectable LLM-style responses.
4. Text Experiment Board: document a prompt test or default test.

## Investigation Prompt

Run the same starting phrase at low and high temperature. Then compare greedy decoding with sampling. What changed in the output, and what did not change about the mechanism?

## Low-AI / No-AI Pathway

Participants can use only the visualizer, ELIZA, and pre-generated comparison examples. They do not need to log into or prompt a live LLM.

## Reflection Questions

- What did the human decide?
- What did the machine predict or complete?
- What made the output feel fluent?
- Where could fluency be mistaken for understanding?
- What classroom adaptation would keep the mechanism visible?
