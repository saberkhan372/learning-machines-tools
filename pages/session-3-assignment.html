Keep one claim moving

Six ways to continue after the core sessions. Every route ends with the same compact evidence package: one observed change, one careful claim, one next test, and one responsible-use boundary.

§ A · The shared deliverable

small enough to finish
At frame/time ___, ___ changed after ___. This suggests ___, although we would need ___ to test that explanation. I would / would not use this output for ___ because ___.

§ B · Choose one route

all count as full participation
Studio artifact · Featured

Bring one evidence-backed artifact

Bring a tool, experiment, classroom activity, creative work, critique, or question set to the optional Studio / Showcase. The artifact can be rough. Pair it with one visible piece of evidence and the boundary of what that evidence lets you claim.

~45–90 min · any form · no slides required

Default · Frozen example

Find the first exact break

Use a frozen clip or frame strip. Choose one relation—identity, object shape, text, camera, lighting, or motion—and mark the first frame where it changes. Describe what you can observe before offering any explanation.

~15–25 min · no account or generation required

Compare · Reference condition

Run a controlled A/B sequence test

Build one previous-frame run and one fixed-anchor run in Temporal Telephone, beginning from the same first frame. Compare the first visible break in each. State what the activity reveals about reference conditions—and what it cannot prove about a real generator.

~25–35 min · two saved runs and one bounded claim

Verify · Provenance

Design a video-verification checklist

Start with a clip that looks convincing or strange. Build a checklist that goes beyond visual glitches: source context, corroboration, provenance records, content credentials, editing history, and known compression. Keep verification and consent as separate questions.

~30–45 min · one practical checklist

Teach · Adapt

Make Temporal Telephone work for your room

Design a paper, whiteboard, or browser version for your learners. Include the Run A / Run B comparison, a prompt for naming one precise failure, and the caveat that the activity is a simplified reference experiment—not a replica of one universal video architecture.

~45–60 min · one-page protocol or rough lesson

§ C · What to bring

one thing you can point to

For the optional Studio / Showcase, bring the artifact itself or a link, screenshot, frozen frame strip, worksheet row, checklist, system map, or lesson draft. It does not need to be polished.

Bring the evidence with it: the exact frame, timestamp, comparison, or source that supports your observation. A strong contribution also says what evidence would still be needed to test the explanation.
